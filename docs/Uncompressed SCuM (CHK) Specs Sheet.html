Unit

UNIT ID Tag

"UNIT"

String

UNIT Length

Number of Units * 36

N/A

UNIT Section

N/A

1 Unit Data Group/Unit

Actual size of UNIT section will vary based on number of units. My offset values assume only 1 unit exists. Each unit data group contains 36 bytes of information in the following format:
- Initial Angle (Integer)
- Unknown Constant (Integer)
- X-Coordinate (Integer) - Actual Coordinate * 32
- Y-Coordinate (Integer) - Actual Coordinate * 32
- Unit Type (Integer) - See Unit List
- Unknown (1 Integer or 2 Bytes) - Constant(s)
- Unknown (1 Integer)
- Unknown (1 Integer)
- Player (Byte) - See Player List
- Hit Point % (Byte)
- Shield Point % (Byte)
- Energy % (Byte)
- Resource Amount (Long)
- In Hanger Amount (Integer)
- Flags (Byte) - Flags by bit# below:
-- 0 - Cloaked
-- 1 - Unused
-- 2 - Burrowed
-- 3 - In Transit
-- 4 - Hallucinated
-- 5 - Invincible
-- 6 - Unused
-- 7 - Unused
- Unknown Bytes (Remaining 0's)

Fog of War

MASK ID Tag

"MASK"

String

MASK Length

X-Size * Y Size

Long

MASK Section

N/A

X-Size * Y-Size Bytes (1 Byte/ Tile)

Fog of War Section values:
Each Byte has the Fog of War values stored for the first 8 players.
Players #9-12 do not have Fog of War enabled.
The Byte of data / Tile stores 8 bits of information:
If the value = 0, then ALL Players can see that tile.
Each bit, if turned off, means that fog of war is disabled
Bit #0: Player #1
Bit #1: Player #2
Bit #2: Player #3
Bit #3: Player #4
Bit #4: Player #5
Bit #5: Player #6
Bit #6: Player #7
Bit #7: Player #8

Strings

STR ID Tag

"STR "

String

STR Length

????

Length

STR Section

N/A

Offset Table Length (2048) + Length of valid strings

String section format:
Max Number of strings: 1024
Offset Tables - 1024 Offsets & Type Markers
Strings - Null Character (0) terminated strings. First string is always an Offset Table member, pointing to the first real string, a null character.
The second string, if it exists, is the SCuM Title.
The third string, if it exists, is the Description.
The fourth string, if it exists, is the default force name?

Forces

Forces ID Tag

"FORC"

String

Forces Length

????

Long

Forces Section

N/A

12 Bytes (1 Byte/Player) + 4 Unknown Bytes + 32 Bits (4 Bytes)

Forces section format:
12 * Force Number (Byte) (1 Number/Player)
Unknown Section (4 Bytes, 2 Integers, or 1 Long)
4 * Flags Section (8 Bits) (1 Section/Group):
Bit #1 = Nothing
Bit #2 = Nothing
Bit #3 = Nothing
Bit #4 = Nothing
Bit #5 = Shared Vision
Bit #6 = Allied Victory
Bit #7 = Allies
Bit #8 = Random Start Location

Wav Audio

Wav ID Tag

"WAV "

String

Wav Length

4048

Long

Wav Section

N/A

4048 Bytes, 2024 Integers, or 1012 Longs

Wav section format:
The values here refer to the strings in the String (STR) section. These indexes are then used to locate the filenames of the WAVs stored with the SCuM, or in the same directory as the uncompressed SCuM (CHK) file. (I think)
Thanks to Sean Mims for this tip.
