Number of Frames

Integer

Gfx are animations and each frame is a picture from that animation, but I'm sure you knew that :)

Max Width

Integer

If the frames were all to be laid on top of each other, they would all fit inside the rectangle with dimensions of the max width and the max height, or the "max; size rectangle."

Max Height

Frame Overview (repeated as many times as there are frames)

X-Adjust Value

Byte

This is the number added to the "x; coordinate" of every pixel in the frame to get that pixel's position in the "max; size rectangle."

Y-Adjust Value

Byte

This is the number added to the "y; coordinate" of every pixel in the frame to get that pixel's position in the "max; size rectangle."

Row Data (example at bottom)

First Byte of Row Data

Byte

The byte minus 128 = number of transparent pixels at the beginning of the Row

Middle Part of Row Data

 

Expanded Below

Last Byte of Row Data

Byte

Last Byte of Row minus 128 = number of transparent pixels at end of the row

Middle Part of Row Data

Color Number Label

Byte

Tells the number of bytes directly after it that are each a pixel and a color number in the palette

Single Pixels

One byte each

Each byte is the number of the color in the palette that makes up it's pixel

Repeating Colors in a Row

2 Bytes

(Optional) Takes two bytes. 1st byte minus 64 equals the number of times the color repeats. 2nd byte is the color number in the palette. "Color; Number Labels" cannot point to them, and after the two bytes, there must be another color number label for any single colored pixels after them (if any exist)

Transparent Pixel

Byte

(Optional) The color number 129, but it's treated like a repeating color, in that it "ColorNumber; Labels" cannot include them, and there must be a new "Color; Number Label" for any single colored pixels after it.

Example of Row Data

Actual Numbers

Name

Explanation

133

First Byte of Row Data

133 minus 128 equals 5, so there are 5 transparent pixels before any other pixels.

3

Color Number Label

The number is 3, so the next 3 numbers are the first three pixels of the row and the numbers can be directly translated into colors.

101

Single Pixel

The first pixel in the row is the 101st color in the palette.

104

Single Pixel

The second pixel in the row is the 104th color in the palette.

110

Single Pixel

The third pixel in the row is the 110th color in the palette.

70

First Byte of "Repeating; Pixels in a Row"

70 minus 64 = 6. So the next color repeats 6 times.

111

Second Byte of "Repeating; Pixels in a Row"

The 4th through the 9th pixels in the row are the 111th color in the palette.

3

Color Number Label

The number is 3, so the next 3 numbers are the 10th through 12th pixels of the row and the numbers can be directly translated into colors.

110

Single Pixel

The 10th pixel in the row is the 110th color in the palette.

104

Single Pixel

The 11th pixel in the row is the 104th color in the palette.

101

Single Pixel

The 12th pixel in the row is the 101st color in the palette.

129

Transparent Pixel

Since the three pixels that the "Color; Number Label" showed have already been written, this is interpreted as a transparent pixel, instead of the 129th color in the palette. So the 13th pixel in the row is transparent.

129

Transparent Pixel

The 14th pixel in the row is transparent.

1

Color Number Label

The next byte is the color of the next pixel

101

Single Pixel

The 15th pixel in the row is the 101st color in the palette.

133

Last Byte of Row Data

133 minus 128 equals 5, so there are 5 transparent pixels after the 15th pixel.
